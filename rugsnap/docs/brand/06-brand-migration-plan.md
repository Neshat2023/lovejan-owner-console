# 06 — Brand Migration Plan (Trigger-Gated)

**Date:** 2026-07-23 · **Status:** Dormant until the Bible §2.2 trigger fires (vertical #2 launch or first institutional raise). Nothing here executes today. RugSnap remains the sole public brand.

## Phase M0 — Preparation (now; zero public change)
- Owner sets acquisition budget (decision D6); counsel knockout search on the doc-05 shortlist (bundle with D7 legal engagement).
- Engineering guardrail already in force: no "rug" assumptions in schema/API/code (Bible rule) — keeps migration a *skin* change, not a rebuild.
- This folder is the single source of truth; any new name idea gets a doc-03 row + doc-04 verification before discussion.

## Phase M1 — Selection (at trigger)
- Re-run live domain checks (data in doc 04 goes stale); complete the doc-05 §11 verification list; acquire parent .com + defensive set within D6.
- Register social handles; file trademark application(s) before any public use.
- Exit criteria: clearance opinion in hand, .com controlled, handles secured.

## Phase M2 — Soft endorsement (weeks after M1)
- Landing pages gain "RugSnap — by <Parent>" endorsement line; legal/ToS entities updated to parent brand naming where applicable.
- No app renames, no domain switches; measure: zero drop in conversion/SEO baselines.
- Rollback: remove endorsement line (one deploy).

## Phase M3 — Platform surfaces under parent (vertical #2 launch)
- New vertical launches inside the platform app/site under the parent brand as a category (never a new Snap-named app — doc 02).
- Parent domain hosts the platform (marketplace, passports); rugsnap.com/.app remain as the rugs front door, 301-preserving SEO equity; sitemap/canonical migration checklist executed (staged 301s, Search Console change-of-address per host, OG re-verification).
- Rollback: parent domain can 301 back to rugsnap.app until DNS/SEO issues resolve.

## Phase M4 — Convergence (12–24 months post-M3, metric-gated)
- When parent branded-search volume ≥ RugSnap's, consolidate app-store listing under parent (single app, categories inside); RugSnap persists as the rugs category brand/landing indefinitely — it is equity, not debt.
- Gate metrics: branded-search crossover, direct-traffic share, retention parity for users acquired under each brand.

## Standing risks & mitigations
- **SEO loss in domain moves:** staged 301s, no simultaneous URL-structure changes, 6-month overlap monitoring.
- **Community alienation:** RugSnap never disappears; Farsi communications lead with continuity («همان راگ‌اسنپ، حالا بخشی از …»).
- **App-store review friction on rename:** rename at a version boundary with unchanged bundle ID; IAP products carry over.
- **Paying twice for naming:** the trigger discipline exists precisely so the umbrella is bought once, with revenue, not twice on speculation.
